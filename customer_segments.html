Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

However, we can quantify the "goodness" of a clustering by calculating each data point's silhouette coefficient. The silhouette coefficient for a data point measures how similar it is to its assigned cluster from -1 (dissimilar) to 1 (similar). Question 10¶

Companies often run A/B tests when making small changes to their products or services. If the wholesale distributor wanted to change its delivery service from 5 days a week to 3 days a week, how would you use the structure of the data to help them decide on a group of customers to test?
Hint: Would such a change in the delivery service affect all customers equally? How could the distributor identify who it affects the most?

Answer:

The wholesale distributor should test the 3-day service on two samples of equivalent size, one taken from Cluster 0 and one taken from Cluster 1. We will call such samples "treatment" groups. By doing so, he would be able to verify how each customer segment reacts as there is no way to know a priori if both segments would be happy with it, if both segments would be unhappy with it or if one would be happy and the other unhappy, along with which of the two would be the happy one and which the unhappy one.

The wholesale distributor should also monitor two other samples of equivalent size, again one taken from each cluster. We will call such samples "control" groups. The purpose of monitoring control groups is to verify that any change in customer behaviour is reasonably likely to be ascribable to the change in service. If both treatment and control groups showed similar reactions in customer behaviour, then it would seem reasonable not to link the change in customer behaviour with the change in service, because the change in behaviour would be more likely to be explained by the some other independent factor. On the contrary, if the treatment group shows some change in customer behaviour that is not observed in the control group, then it would appear reasonable to link the change in customer behaviour with the change in service.
